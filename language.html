the language

where application is currying and everything is a function

ai is a dynamically typed lisp superset with curried evaluation, fast generics, and optional infix/prefix notation. the front page is the brass tacks — the shell and the reference; this page is the why. every demo here runs live in the front page shell, and every law has its proof-by-assertion in test/spec.l.

everything is a function

every value is theoretically a one-argument total function of global type. application associates left — (f x y) is ((f x) y) — so multi-argument calls are just currying, and partial application is not a feature, it is what application is. at zero operands (f) is f: there is nothing to apply, so nothing happens.

numbers act as their church numerals — n applied is n-fold iteration, so a number applied to a number is exponentiation, and the numerals you compute with and the numerals you iterate with agree by construction. data self-applies too: a string indexes its bytes, a book is its own lookup.

(0 5)              ; 1        0 is const-1
(1 5)              ; 5        1 is the identity
(0 0 'x)           ; x        const of const is id
(3 2)              ; 8        (n x) = xⁿ
(2 3 4)            ; 262144   the tower 4^(3^2)
(3 3 3)            ; 7625597484987    tetration is the tower with one base
(8 = 3 2 (+ 1) 0)  ; 1        iterating (+ 1) agrees with the arithmetic
(map (+ 1) '(1 2 3)) ; (2 3 4)  currying is partial application
((/ 1 2) 9)        ; 3.0      fractional powers are roots

three special forms

: is letrec* and sequence, ? is cond, \ is lambda (and, at one operand, quote). everything else — the whole rest of the language — is a function call.

(: a 1 b 2 (a + b))          ; 3     bind in source order; the last form is the result
(? 0 'no (1 < 2) 'yes 'else) ; yes   test/result pairs, then an else
((\ x y (x + y)) 3 4)        ; 7     args then body, auto-curried

the trio has a pedigree: they are exactly the three forms of steele & sussman's lambda: the ultimate imperative (1976), where they are called LABELS, IF, and LAMBDA — the kernel the lambda papers showed could swallow every imperative construct. they are also the three generators of plotkin's PCF, the canonical minimal calculus of general recursive computation: abstraction (\), case analysis (?), and fixpoint (: — letrec is fix in friendly clothing).

in pure lambda calculus two of the three are redundant — church booleans make conditionals mere application, and the Y combinator conjures recursion from \ alone. what the trio minimally generates here is something more honest about evaluation: they are precisely the forms that must not evaluate all their operands. \ defers its body, ? must not run the untaken branch, : lets a binding refer to what isn't pinned yet. a form that evaluates everything can be a function — and in ai, is — so the special forms are the minimal set of evaluation-order exceptions: one each for binding, choice, and time.

true and false

false is nothing: whatever nets ≤ 0. the net is the content measure — a number is its own net, text sums its bytes, a list or array sums its elements, negatives cancelling. every value wears a color by its net's sign — green positive, red negative, blue the zero floor — and truth is exactly green. there is no truthy and falsy: the truth bit of x is !!$x, and every nothing (0, (), "", an empty or cancelled list) sits on the same blue floor.

!0  !""  !()       ; 1      every nothing is false
!'(-2 1)           ; 1      red at any rank is false too
!5                 ; 0      positive green is true
($ '(1 2 3))       ; 6      $ sums the nets, then clamps once
($ -7)             ; 0      red saturates to the floor

equality is exact

= compares functions by alpha-equivalence and structure — two lambdas that differ only in the names of their binders are the same value. and it is exact everywhere else too: identities hold on the nose or not at all, never approximately.

((\ x x) = (\ y y))            ; 1    alpha-equivalent lambdas compare equal
((\ x y (x + y)) = (\ a b (a + b))) ; 1
((log -1) = i * pi)            ; 1    euler, in the exact direction
(i = ((/ 1 2) -1))             ; 1    the principal root, on the nose
(2 i)                          ; ~(-1.0 0.0)    i² = -1

the ultimate goto

the lambda papers' companion thesis — steele's lambda: the ultimate goto — argued that a procedure call is a jump wearing a return address it doesn't need. ai's vm takes that literally and makes it law: the interpreter is tail-threaded, every operation tail-jumps to the next, and calls never return (make vmret checks it). scheduling is mandatory and cooperative, riding the same discipline; call-cc is a one-shot escape through it.

(call-cc (\ k (+ 1 (k 41))))   ; 41   the escape fires, the (+ 1) never runs

the rest of the facts

  • infix and prefix notation are strictly sugar — they desugar to lisp before anything else looks at the source
  • the numeric tower: charms overflow into bigs, gems and twins for the smooth and rotating kinds, and galaxies broadcast over all of them
  • the usual lisp stuff: lists, symbols, quote, macros, call/cc
  • a strictly optional static dependent type system (in early development) — its kernel is proved sound and its proofs export to rocq and lean
  • a repl with native line editing
  • a self-hosting threaded-code interpreter with a template JIT (amd64/arm64) — see the benchmarks
